jQuery plugin: Route Enter

This plugin is for catching the "Enter" key in your form and routing it elsewhere, usually to another button. Some developers are unable to use separate form tags for each semantic form: ASP.NET WebForms, for instance. Thus, when a user presses Enter in a form the browser clicks the first button on the form by default. routeEnter uses this behavior to its advantage by inserting a hidden button at the beginning of the form to catch these. Extended documentation is in progress.

Examples

No configuration

Will default to the first submit button (input[type=submit]:first) in each section. When the form submits you will see the submitted button in the query string.

{% highlight js linenos %}$("#noConfigExample .register, #noConfigExample .login").routeEnter();{% endhighlight %}

Documentation

Options

Name Type Default Notes
button Selector, Element, jQuery "input[type=submit]:first" Not required
formDataKey String "enterRouted" Not required. Marks a form as initialized for routing. You shouldn't need to do anything with this.
fields Selector, Element, jQuery ":input" Not required. Determines which fields inside of the section that can "focus" the section. You shouldn't need to do anything with this.
buttonEvent String "click" Not required. Change this if you want something other than "click" to be triggered on the section's button.
targetForm Selector, Element, jQuery $("form:first") Not required. If you have multiple forms on a page you may need to set this.
router Object See below Not required. Options for specifying attributes and styles for the router button.
router.attr Object {type:"submit", id:"enterRouter", name:"enterRouter"} Not required. Router button attributes.
router.css Object { position:'absolute', left:'-2000px' } Not required. Router button styles. Using absolute positioning to move the button offscreen because IE (at least) will only use the first visible button.
